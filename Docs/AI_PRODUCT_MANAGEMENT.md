# AI Product Management

> Full process framework document. This file defines the workflow, document lifecycle, naming conventions, parallel planning patterns, and self-review tooling for building products with AI coding agents.
>
> **Source:** Shared by operator at project inception (2026-03-22). The full document was provided as an attachment in the initial setup conversation.
>
> **Key principles this project follows:**
> - Many small releases over few large ones
> - Every chat produces a document
> - Research → Design → Build+Test → Release → Reference doc update
> - The AI does the work; the human does the judgment
> - Capture mistakes so the agent learns (CLAUDE.md Known Mistakes section)
> - Build observable by default
>
> **Document categories:**
> 1. Reference docs (`Docs/` top-level) — current system state
> 2. Design docs (`Docs/Design/`) — proposals & specs
> 3. Research docs (`Docs/Research/`) — persistent findings
> 4. Testing docs (`Docs/Testing/`) — test logs & analysis
>
> **Session logs:**
> - `Docs/REQUEST_LOG.md` — user intent
> - `Docs/DEVELOPMENT_LOG.md` — technical decisions
> - `CHANGELOG.md` — version changes
> - `ROADMAP.md` — release status
>
> **Versioning:** Sequential (v0.1.0, v0.1.1...), one version = one commit, small focused releases.

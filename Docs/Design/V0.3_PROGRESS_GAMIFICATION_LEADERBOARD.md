> Status: **Draft**
> Target version: v0.3.0
> Depends on: v0.2.0 (Supabase auth + Vercel deployment)
> Created: 2026-03-22

---

## Overview

This release adds the three interconnected systems that make learning sticky and motivating:

1. **Progress Tracking** — precise per-user record of what has been learned, practiced, and retained
2. **Gamification** — XP, streaks, badges, and levels that reward consistency and achievement
3. **Leaderboard** — real-time ranking of all learners to drive friendly competition

All three depend on Supabase for persistence and realtime updates.

---

## 1. Progress Tracking

### Goal
Every time a user logs in, the app knows exactly where they are: which lessons are complete, which words/phrases need review, and what to do next.

### Data Model

#### `user_progress` table
| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| user_id | uuid | FK → auth.users |
| lesson_id | text | Identifier for the lesson/unit |
| status | enum | `not_started`, `in_progress`, `complete` |
| score | int | Last attempt score (0–100) |
| attempts | int | Number of attempts |
| last_practiced_at | timestamp | For spaced repetition scheduling |
| next_review_at | timestamp | Calculated next review date |
| created_at | timestamp | |
| updated_at | timestamp | |

#### `vocabulary_retention` table
| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| user_id | uuid | FK → auth.users |
| word_id | text | Vocabulary item identifier |
| ease_factor | float | SM-2 spaced repetition ease factor |
| interval_days | int | Current review interval |
| repetitions | int | Consecutive correct answers |
| due_at | timestamp | Next review due date |
| last_reviewed_at | timestamp | |

### Spaced Repetition
Use a simplified SM-2 algorithm to schedule vocabulary reviews:
- Correct answer → interval increases (1 → 3 → 7 → 14 → 30 days)
- Wrong answer → reset interval, reduce ease factor
- Due items surface at the start of each session

### "Where I Left Off" Session Restore
On login, the app queries:
1. Most recent `in_progress` lesson → resume it
2. Any `next_review_at` past due → show review queue
3. Next `not_started` lesson in the curriculum sequence

## 2. Gamification

### Goal
Reward daily engagement and mastery milestones. Make progress feel tangible and fun.

### XP System

| Action | XP Earned |
|--------|-----------|
| Complete a lesson | 50 XP |
| Perfect score on a lesson | +25 XP bonus |
| Complete a vocabulary review | 10 XP |
| Daily streak (per day) | 20 XP |
| First attempt at a lesson | +10 XP bonus |
| Complete a chapter (5 lessons) | 100 XP bonus |

### Levels

| Level | Name | XP Required |
|-------|------|-------------|
| 1 | Beginner | 0 |
| 2 | Explorer | 200 |
| 3 | Adventurer | 500 |
| 4 | Traveller | 1,000 |
| 5 | Local | 2,000 |
| 6 | Speaker | 4,000 |
| 7 | Fluent | 7,500 |
| 8 | Native Heart | 12,000 |

### Streaks
- A streak is maintained by completing at least one lesson or review per calendar day
- Streak freeze: one missed day can be forgiven (single-use per week, earned at level 3+)
- Displayed prominently in the dashboard (e.g. "🔥 12-day streak")

### Badges

| Badge | Trigger |
|-------|---------|
| First Step | Complete first lesson |
| Hot Streak | 7-day streak |
| On Fire | 30-day streak |
| Perfectionist | 5 perfect scores in a row |
| Vocabulary Builder | 100 words mastered |
| Chapter Champion | Complete a full chapter |
| Speed Learner | Complete 3 lessons in one day |
| Comeback Kid | Return after 7+ days away |

### Data Model

#### `user_gamification` table
| Column | Type | Description |
|--------|------|-------------|
| user_id | uuid | PK + FK → auth.users |
| xp_total | int | Cumulative XP |
| level | int | Current level (derived from xp_total) |
| streak_current | int | Current streak in days |
| streak_longest | int | All-time longest streak |
| streak_freeze_available | bool | Whether a streak freeze is available |
| last_active_date | date | Last day with activity (for streak calc) |
| badges | text[] | Array of earned badge IDs |
| updated_at | timestamp | |

## 3. Leaderboard

### Goal
Show learners how they rank against others — weekly and all-time — to drive friendly competition.

### Views

| View | Description |
|------|-------------|
| **Weekly** | XP earned in the current calendar week — resets Monday |
| **All-Time** | Total XP across all time |
| **My Rank** | User's current rank highlighted, with neighbors shown |

### Display Fields (per row)
- Rank (#1, #2, #3…)
- Avatar / initials
- Display name (user-chosen, not email)
- Level badge
- XP (weekly or all-time depending on view)
- Current streak

### Realtime Updates
Use Supabase Realtime subscriptions so the leaderboard updates live as other users earn XP (no manual refresh needed).

### Privacy
- Users choose a display name on signup — real name / email never shown publicly
- Option to appear as "Anonymous" on the leaderboard

### Data Model

#### `leaderboard_weekly` (Supabase view or materialized)
Computed from `user_gamification` + `xp_events` filtered to current week.

#### `xp_events` table
| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| user_id | uuid | FK → auth.users |
| amount | int | XP earned |
| reason | text | e.g. `lesson_complete`, `streak_bonus` |
| earned_at | timestamp | For weekly aggregation |
